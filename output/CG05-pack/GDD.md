## §1 项目定位

沙滩主题 Mines 博弈小游戏:玩家在 5×5 沙滩盘面逐格翻沙块,安全格翻出金币、兑付倍率沿雷数对应的倍率梯上爬,可随时收款离场;翻到水雷即本局失败;翻完全部安全格自动按该雷数封顶倍率结算(Grand Prize)——`crash-step` 交互踩在 `grid` 盘面上。单人离散对局,无对局倒计时、无多人。竖屏移动优先。

## §2 游戏流程与状态机

| 状态 | 标识符 | 说明 |
|---|---|---|
| 启动加载 | `loading` | logo + 进度条(boot/资产/配置/socket 四段聚合);完成出「Game Start」钮 |
| 待机下注 | `betting` | 选注额+雷数;盘面全盖;第三钮=REFRESH 面(**入场即此态**) |
| 游戏中 | `playing` | REFRESH 按下即下注扣款并开局;逐格翻开,任意时机可收款;首翻后第三钮变 CASH OUT |
| 爆雷 | `busted` | 翻中水雷,本局清零;全盘揭示 + 格内爆炸 + 全屏屏裂震动 |
| 局结算 | `round_settled` | 盘面保持揭示(玩家翻的全亮、其余半透明);**无自动回投注**,等玩家再按 REFRESH 开下一局 |

`cash_out` 与 `perfect_clear` 为**瞬态事件**(同一信号 `NET_MY_CASHOUT`,以 `isUltimate` 区分),不驻留状态。

```
loading ──就绪+Game Start──► betting ──REFRESH(=下注+开局)──► playing
   ▲                            ▲                              │
   │                            │              ┌──[翻格 safe]──┤ (倍率沿梯上爬,可随时 CASH OUT)
   │                            │              ├──[翻格 mine]──► busted ──────┐
   │                            │              ├──[CASH OUT]──► (YOU WIN 4s) ─┤
   │                            │              └──[翻满安全格]─► Grand Prize ──┤
   │                            └────────(玩家按 REFRESH,盘面才重盖)◄─ round_settled
   └─(异常)── maintenance / 错误弹窗
```

- 收款/爆雷/通关后盘面**保持揭示**直到下一次 REFRESH——没有结算读秒或 Play Again 按钮;YOU WIN 横幅自身 4 秒后淡出,但状态不回。
- 覆层(settings / language / guide / history / warning / toast / 退出确认)叠加在主状态上,不进主状态机;settings 全屏覆层打开时吸收全部输入。
- 标准局:选注额(弹层选档)→ 选雷数(3~24 弹层)→ 按 REFRESH 锁注 → 逐格点沙块(免费,一次一发)→ 任意格 CASH OUT 落袋 / 翻满自动 Grand Prize / 爆雷清零 → 盘面揭示展示 → 再按 REFRESH 开新局。锁注后不可改注额与雷数(选择器弹层被禁开)。

## §3 核心玩法

| 参数 | 标识符 | 规格 | 默认值 |
|---|---|---|---|
| 下注金额 | `bet_amount` | 弹层选档(服务端 /bet/config 下发 **2~6 档** betList + defaultBet;无手动输入);兜底档 USD [0.2,0.5,1,3,5,10] | 服务端 defaultBet(兜底:USD 1 / 其他币 0.1) |
| 地雷数 | `mine_count` | 整数 3~24,弹层 4 列×6 行数字矩阵(两个空格) | 4 |
| 盘面大小 | `grid_size` | 固定 5×5 = 25 格 | 25 |
| 安全格 | `tile_safe` | 数量 = 25 − mine_count,翻开显示金币、倍率沿梯推进 | — |
| 地雷格 | `tile_mine` | 数量 = mine_count,翻开即本局失败 | — |
| 当前倍率 | `current_multiplier` | **查表制**:`MULT[mine_count][已翻安全格数−1]`(服务端倍率梯权威,见 §4);倍率条高亮当前档 | 未开局 0 |
| 开局按钮 | `refresh` | 第三钮 REFRESH 面=重盖盘面+下注(**扣款点**);翻格本身免费 | — |
| 翻格节流 | `go_throttle` | 单发飞行门(一次一格)+ 600ms 节流 | — |

玩法要点:

- Cash Out:`playing` 中首翻后任意时机,`payout = bet_amount × MULT[m][k]`,服务端回执 `payoutAmount` 为权威;收款请求期间按钮置灰、双击被 pending 门拦截。
- 完美通关:翻完全部安全格,服务端在最后一格回执直接标记完局(`gameOver=true`),按该雷数封顶倍率自动兑付并触发 Grand Prize 演出;余额更新延迟到演出弹出瞬间才入账(防剧透)。
- 爆雷:翻中雷,回执附全部雷位,全盘揭示(玩家翻的全亮,其余 50% 透明,踩中的雷带爆炸特效),损失本金。
- 揭示保持:局末盘面不自动重盖——是「上一局的证据」,直到玩家主动开新局。

## §4 数学框架

倍率表为**服务端权威**:连接后 `client:routes` 一次性下发全部 22 条倍率梯(雷数 3~24 各一条),开局回执再附本局梯;前端内置同值副本仅作离线兜底。实测表(前端副本,与后端逐档一致交付):

`MULT[m][k]` = 雷数 m、已翻 k 个安全格的兑付倍率;每行末档 = 该雷数**满盘封顶**(自动兑付值)。

| 雷数区间 | 封顶倍率 | 首档示例(k=1) | 档数 |
|---|---|---|---|
| 3 ~ 5 | **20.00x** | 1.11 / 1.16 / 1.22 | 22 / 21 / 20 |
| 6 ~ 10 | **80.00x** | 1.28 / 1.35 / 1.43 / 1.52 / 1.62 | 19~15 |
| 11 ~ 15 | **300.00x** | 1.74 / 1.88 / 2.03 / 2.22 / 2.44 | 14~10 |
| 16 ~ 20 | **800.00x** | 2.71 / 3.05 / 3.48 / 4.06 / 4.88 | 9~5 |
| 21 ~ 23 | **200.00x** | 6.09 / 8.12 / 12.19 | 4 / 3 / 2 |
| 24 | **24.38x** | 24.38(单档即封顶) | 1 |

逐档样例(全量 22 行随交付物 `elements` 附):

| 雷数 \ 已翻格 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|
| 3 | 1.11 | 1.27 | 1.46 | 1.68 | 1.92 | 2.21 |
| 4(默认) | 1.16 | 1.34 | 1.54 | 1.78 | 2.05 | 2.36 |
| 10 | 1.62 | 2.15 | 2.84 | 3.75 | 4.95 | 6.53 |
| 20 | 4.88 | 17.45 | 62.45 | 223.52 | 800.00 | — |
| 24 | 24.38 | — | — | — | — | — |

- **RTP 复算**:公平倍率 `fair(k)=C(25,k)/C(25−m,k)`;实测首段各档 ≈ fair×0.975(例:m=24,k=1 → 25×0.975=24.375→24.38;m=3,k=1 → 25/22×0.975≈1.108→1.11)。深档被封顶截断(m=3 的公平尾档远超 20x),该段有效 RTP 低于 0.975——封顶策略与整体 RTP 的最终口径以数学组/服务端为准(§6)。
- **口径矛盾(实装事实)**:玩法说明页文案写 “Max Win: x10,000 / Max Payout = Max Win × Max Bet”,而倍率表最大封顶 800.00x。两处同时存在于实装,服务端运营口径待定(§6)。
- 显示格式:倍率 `toFixed(2)+"X"`;金额按币种小数位(USD 2 / CNY 1 / PHP 0 / BRL 2,其余 2)。

## §5 系统边界(美术相关)

| 边界 | 触发 | 是否产生 UI 素材 | 说明 |
|---|---|---|---|
| 余额不足 | 注额 > 余额(REFRESH 时校验) | yes | Toast「Insufficient balance…」,下注阻断 |
| 请求失败 | 下注/翻格/收款 ACK 超时 **8s** | yes | Toast「Server did not respond. Please retry.」(1s)+按钮回弹再试 |
| 断线自动兑付 | `playing` 中断线 | yes | 常驻 Toast 重连中+**15s 倒数**(每秒 tick);归零由服务端自动兑付;重连成功 3s Toast |
| 闲置超时 | 服务端 `idle-timeout` 告警 | yes | Warning 弹窗(带倒数文案)→ 自动兑付;确认钮=手动重连 |
| 重连耗尽 | socket.io 自动重连 ≤10 次(退避 1~5s)全败 | yes | Warning 弹窗「Connection lost…」;确认=再试一次,10s watchdog 兜底整页刷新 |
| 会话过期 | token 失效 / `server:game:expired` | yes | 常驻/3s Toast「Session expired…」;进行中的局防御性收局 |
| 单一 session | 同账号他端进入 | yes | 服务端 `sso-disconnected` 弹窗踢出;该断线**不进** 15s 倒数 |
| 收款进行中 | Cash Out 未回执 | yes | 收款钮置灰,盘面与选择器锁定,直到回执 |
| 会话续局 | 断线/刷新后重进且局未结 | yes | 快照恢复:重放已翻格、锁 UI 为 CASH OUT 态、倍率条回到当前档(无重播动画) |
| 维护/封禁/币种不支持 | 平台异常态 | yes | DOM 维护屏(羊皮纸面板+右上退出钮,仅 backURL 存在时显示) |
