§1 项目定位

麻将招财主题的视频 slot：符号为麻将牌面，中奖走 Tumble（级联连消）+「连莊」递增倍率玩法——盘面消除后剩余符号塌落、空位再填充，同一局内每连消一轮，全局倍率沿档位递增，直至无新的中奖组合。金色麻将是本作标志机制：金符参与中奖后原位转化为「百搭」。

    • §2 游戏流程与状态机

    外壳层（React）负责 Loading / 异常 / 桥接，Unity 层负责基础转、Tumble 结算、连莊倍率与 Free Spin。状态标识符 snake_case。

    状态标识符说明
    加载loadingbootstrap：背景 + Loading 进度，等待登录 + 资产 + Unity GameReady
    入场entry入场页（背景 + 标题 + 开始按钮），点击进入主游戏
    主游戏main_game转轴就绪，等待投注 / Spin
    转轴判定spin_resolve一次 Spin 落定后判定中奖组合（左起连列 ways，见 §3）
    级联消除tumble_cascade中奖符号消除（金符原位转百搭）→ 塌落 → 填充；每轮倍率档 +1（连莊）
    中奖演出win_present每轮结算横幅 + 达档触发 大奖 / 巨奖 / 超级巨奖 演出层
    免费旋转公告free_spin_announce最终盘面胡（Scatter）≥3 触发；公告 / 引导 / 开始确认屏
    免费旋转进行free_spin_playFree Spin 循环：倍率档翻倍梯（×2/×4/×6/×10），reel3 全金
    免费旋转结束free_spin_end结束确认屏（累积奖金），回主游戏
    维护 / 异常maintenance外壳 exception 归一态（全屏暗框）

    ```

    loading ─► entry ─► main_game ──on_spin──► spin_resolve ──有中奖──► tumble_cascade ──┐

    ▲ │ │ (循环;倍率+1) │

    │ │无中奖 └──────────────┘

    │ ▼ 每轮 │

    │ main_game ◄── win_present ◄──────────┘
    ```

    标准局流程：设定投注额 → Spin → 落定后按「左起连列」判定各符号 ways → 有奖则逐轮级联（消除 / 金符转百搭 / 塌落 / 填充），倍率档随轮次递增，赢分逐轮累计 → 最终盘面统计胡数量，≥3 触发 Free Spin → 免费局倍率梯翻倍、第 3 轴全金。

    §3 核心玩法

    参数标识符规格默认值
    盘面几何board_geometry5 列异形：边列 4 格、中三列 5 格（4-5-5-5-4，共 23 格）；顶部另有半格预览位实测定稿
    中奖判定win_evaluation左起连列 ways：符号须自第 1 列起连续出现 ≥3 列，ways=各列命中数乘积；全盘最大 4·5·5·5·4=2000 ways实测定稿
    级联消除tumble_cascade中奖符号消除 → 塌落 → 按轴带填充，循环至无新中奖实测定稿
    连莊倍率streak_multiplier同局级联轮次驱动全局倍率：基础局 ×1/×2/×3/×5（第 4 轮起封顶）；免费局 ×2/×4/×6/×10实测定稿
    金符转百搭gold_to_wild金色麻将仅现于第 2/3/4 列；参与中奖后不消除、原位转化为百搭实测定稿
    百搭wild_baida替代除胡外全部符号；自身无独立赔付、无倍率实测定稿
    胡（Scatter）scatter_hu全列可现；一局最终盘面计数 ≥3 触发免费；不参与派彩实测定稿
    免费旋转free_spin胡 3/4/5/6/7 → 10/12/14/16/18 次；免费中再触发按同表追加（retrigger）；免费局第 3 轴全金（除百搭/胡）实测定稿
