## §1 项目定位

麻将招财主题的视频 slot：符号为麻将牌面，中奖走 Tumble（级联连消）+「连莊」递增倍率玩法——盘面消除后剩余符号塌落、空位再填充，同一局内每连消一轮，全局倍率沿档位递增，直至无新的中奖组合。金色麻将是本作标志机制：金符参与中奖后原位转化为「百搭」。

- 代号：SS03
- interaction：slot
- board：reel（tumble 级联；4-5-5-5-4 异形盘面，见 §3）
- 参考游戏：无（原创麻将 slot；客户端沿用平台共用 Unity-middleware 外壳）
- 屏幕与适配：1080×1920 基准 portrait；landscape 下同一竖版画布居中、两侧由全幅背景延展（实测无独立横版布局）
- 语言：英文(en), 印度语(hi), 印尼文(id), 韩文(ko), 马来文(ms-MY), 泰文(th), 越南文(vi), 简体中文(zh-hans), 繁体中文(zh-hant)（共 9，另有 Shared 非语言共用表；实测 Localization 表即此 9 locale）
- 货币：TBC（见 §6，由运营 / PM 定；金额显示两位小数）
- 目标 RTP：TBC（见 §6，数学组定；支持多套可换数学表按运营配置分配）
- 非目标：不自建钱包（依赖平台 Wallet API）；不含社交分享；不含 jackpot

## §2 游戏流程与状态机

外壳层（React）负责 Loading / 异常 / 桥接，Unity 层负责基础转、Tumble 结算、连莊倍率与 Free Spin。状态标识符 snake_case。

| 状态 | 标识符 | 说明 |
|---|---|---|
| 加载 | `loading` | bootstrap：背景 + Loading 进度，等待登录 + 资产 + Unity GameReady |
| 入场 | `entry` | 入场页（背景 + 标题 + 开始按钮），点击进入主游戏 |
| 主游戏 | `main_game` | 转轴就绪，等待投注 / Spin |
| 转轴判定 | `spin_resolve` | 一次 Spin 落定后判定中奖组合（左起连列 ways，见 §3） |
| 级联消除 | `tumble_cascade` | 中奖符号消除（金符原位转百搭）→ 塌落 → 填充；每轮倍率档 +1（连莊） |
| 中奖演出 | `win_present` | 每轮结算横幅 + 达档触发 大奖 / 巨奖 / 超级巨奖 演出层 |
| 免费旋转公告 | `free_spin_announce` | 最终盘面胡（Scatter）≥3 触发；公告 / 引导 / 开始确认屏 |
| 免费旋转进行 | `free_spin_play` | Free Spin 循环：倍率档翻倍梯（×2/×4/×6/×10），reel3 全金 |
| 免费旋转结束 | `free_spin_end` | 结束确认屏（累积奖金），回主游戏 |
| 维护 / 异常 | `maintenance` | 外壳 exception 归一态（全屏暗框） |

```
loading ─► entry ─► main_game ──on_spin──► spin_resolve ──有中奖──► tumble_cascade ──┐
              ▲                              │                        │ (循环;倍率+1) │
              │                              │无中奖                   └──────────────┘
              │                              ▼                            每轮 │
              │                          main_game ◄── win_present ◄──────────┘
              │                              │ 最终盘面胡≥3
              │                              └──► free_spin_announce ─► free_spin_play ─► free_spin_end ─┐
              └──────────────────────────────────────────────────────────────────────────────────────────┘
loading/main_game ──exception──► maintenance ──(清除)──► loading
main_game ──ExitGame(backURL / history.back)──► [退出]
```

标准局流程：设定投注额 → Spin → 落定后按「左起连列」判定各符号 ways → 有奖则逐轮级联（消除 / 金符转百搭 / 塌落 / 填充），倍率档随轮次递增，赢分逐轮累计 → 最终盘面统计胡数量，≥3 触发 Free Spin → 免费局倍率梯翻倍、第 3 轴全金。

## §3 核心玩法

| 参数 | 标识符 | 规格 | 默认值 |
|---|---|---|---|
| 盘面几何 | `board_geometry` | 5 列异形：边列 4 格、中三列 5 格（4-5-5-5-4，共 23 格）；顶部另有半格预览位 | 实测定稿 |
| 中奖判定 | `win_evaluation` | 左起连列 ways：符号须自第 1 列起连续出现 ≥3 列，ways=各列命中数乘积；全盘最大 4·5·5·5·4=2000 ways | 实测定稿 |
| 级联消除 | `tumble_cascade` | 中奖符号消除 → 塌落 → 按轴带填充，循环至无新中奖 | 实测定稿 |
| 连莊倍率 | `streak_multiplier` | 同局级联轮次驱动全局倍率：基础局 ×1/×2/×3/×5（第 4 轮起封顶）；免费局 ×2/×4/×6/×10 | 实测定稿 |
| 金符转百搭 | `gold_to_wild` | 金色麻将仅现于第 2/3/4 列；参与中奖后不消除、原位转化为百搭 | 实测定稿 |
| 百搭 | `wild_baida` | 替代除胡外全部符号；自身无独立赔付、无倍率 | 实测定稿 |
| 胡（Scatter） | `scatter_hu` | 全列可现；一局**最终盘面**计数 ≥3 触发免费；不参与派彩 | 实测定稿 |
| 免费旋转 | `free_spin` | 胡 3/4/5/6/7 → 10/12/14/16/18 次；免费中再触发按同表追加（retrigger）；免费局第 3 轴全金（除百搭/胡） | 实测定稿 |
| 购买免费 | `buy_free_spins` | 「Buy 10 FG」入口具备（客户端存量），上线口径与定价 TBC | TBC |

## §4 数学框架

派彩以「注额积分」为单位：**Bet Credit = 总注 / 20**。

| 项 | 公式/规格 | 示例值 |
|---|---|---|
| 单组派彩 | `payout = BetCredit × 符号赔率(连列数) × ways × 当轮倍率` | 见下赔付表 |
| 赔付表 | 9 符号 × 3/4/5 连列赔率（金符与同名白符同赔率） | 实测定稿（下表） |
| 连莊倍率 | 基础 ×1/×2/×3/×5；免费 ×2/×4/×6/×10（按级联轮次取档，封顶第 4 档） | 实测定稿 |
| 免费触发 | 胡 3/4/5/6/7 → 10/12/14/16/18 次 | 实测定稿 |
| 目标 RTP / 波动 | 多套可换数学表（转轴带 / 权重按表分配） | TBC |
| 符号权重 / 转轴带 | 服务端数学表交付 | TBC |
| 单局上限 | 服务端按玩家配置有最大派彩封顶（MaxPayout） | 数值 TBC |

赔付表（赔率 × BetCredit；3 / 4 / 5 连列）：

| 符号 | 3 连 | 4 连 | 5 连 |
|---|---|---|---|
| 發 | 10 | 25 | 50 |
| 中 | 8 | 20 | 40 |
| 白 | 6 | 15 | 30 |
| 八萬 | 5 | 10 | 15 |
| 五筒 | 3 | 5 | 12 |
| 五條 | 3 | 5 | 12 |
| 三筒 | 2 | 4 | 10 |
| 二筒 | 1 | 3 | 6 |
| 二條 | 1 | 3 | 6 |

> 金色变体（發~二條各一）赔率与同名白符完全一致；其价值在「中奖后转百搭」。胡 / 百搭无独立赔付行。

## §5 系统边界（美术相关）

| 边界 | 触发 | 是否产生 UI 素材 | 说明 |
|---|---|---|---|
| 断线重连 | 运行中网络中断 | yes | 重连提示 / Toast（"连接服务器中…"）；恢复后回到中断前状态 |
| 闲置登出 | 3 分钟未下注 | yes | 提示弹窗（"3 分钟未旋转已登出"）+ Reload |
| 余额不足 | 下注 > 余额 | yes | 提示弹窗（"余额不足请充值"） |
| 单一 session | 同账号他处登录 | yes | 弹窗提示，禁止双端并行 |
| 外壳异常态 | 平台维护 / 版本 / 货币不支持等 | yes | 全屏纯 CSS 暗框（标题 + 描述 + 可选退出按钮 + 版本号） |
| 语言切换 | Unity 内切语言 | yes | `SetLanguage` 桥同步外壳 locale；标题 / 演出图按语言换图（ADD §3e / §6） |
| 派彩 / 结算失败 | Wallet / 结算返回失败 | yes | 处理中 / 重试提示（文案见共用壳 exceptions，ADD §8） |
