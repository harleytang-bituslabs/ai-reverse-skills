分区重叠 & z 序：GameView 131–1100、PlayerHUD 1100–1920 在 y=1100 处接缝；HUD 顶有 separator_line(金线 1080×16 @1100) + #030204@80% 压暗，露出路尾。难度器展开(L2)会向上弹到 1004，盖住 GameView 底部(L2 在 L1 之上)。overlay/弹窗(L3) z 最高。把②的某内容帧 + ③的某 HUD 帧按此叠放 = 整屏（见画廊④合成屏）。

② 游戏内容层 ContentView（相位驱动 · NET_ROUND_*）

L1 = TopBar(余额/币种/齿轮，常驻 chrome) + GameView(玩法演出，随回合相位切)。每帧只画本层；底部 ghost 区是 PlayerHUD 占位(见画廊③)。GameView 视口里的滚动长卷见画廊④。各相位：betting / fly / crash / win / ultimate(终点) / boarding(登车 cut-in)。

③ 玩家 HUD 层 PlayerHUD（钱包驱动 · WALLET_*）

L2 = 固定框(separator+压暗 + 加减注 BetSwitcher + 5 档快捷投注) + 可换的 betActionStack。仅 PlayState 在变：bet(难度器闭合+紫 PLAY) ↔ cashout(套现+继续)，外加难度器展开。顶部 ghost 区是 ContentView 占位(见画廊②)。注：HUD 布局与游戏相位无关，只随 WALLET_PLAY_STATE 切。

④ 场景长卷 road（GameView 内的滚动内容，真实全高）

属 L1(ContentView/GameView) 的可滚动内容：每难度 = start + middle(×N 无缝循环) + end 沿移动轴向上滚成长卷，同 S 弯换主题皮。绿线=真 centerline.json 中段；红点=站点，站牌挂对侧；蓝框=1080×969 起始视口；车锚屏 y=727。

⑤ 覆盖层 SettingPanel + 弹窗 PopupManager（L3，全屏 overlay）

独立于 L1/L2 的顶层：设置面板(3 tab) + 语言/Toast/断线弹窗。压在游戏之上(z999+)。

⑥ 进场 DOM（React，Pixi 之前/之上）

loading / ready / 异常 / 确认弹窗（React DOM 层，非 Pixi）。独立于 Pixi 三层，全屏 z9999。

⑦ 典型合成屏（②内容 + ③HUD 叠合）

把画廊②的某内容帧 + 画廊③的某 HUD 帧按「① 合成契约」叠成整屏 —— 这是"玩家实际所见"，也是截图验证的锚。常见配对：betting+bet / fly+cashout / win+cashout。

⑨ 滚动路渲染机制（详见画廊④ + GDD 玩法含义）
